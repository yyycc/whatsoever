为什么要读源码。
工作以来很长一段时间，是没有读过源码的，可能偶尔会看一眼，但是不专门也不深入。
有问题么，其实也没啥问题，开发也能照常进行，出了问题，google一下基本都能解决。
那为什么要读源码呢？
我们现在的日常开发其实越来越简单，因为有很多已经封装好的框架、工具，我们在这些基础上进行开发，easy。
照着文档，一步一步搭建，yarn、yarn serve
hey，界面出来了，good。
谁又会去想界面为什么能出来呢。
当写了一句
```js
this.name = 'whatsoever'
```
界面的展示也跟着变了，谁会去想这一句话的背后，执行了多少代码呢。
至少很多很多年，我没有去想过。
想的都是成功了，yeah，失败了，把错误信息复制粘贴检索，解决了，yeah。。。
为什么会失败，为什么能解决，为什么会成功。不知道。
所以为什么要读源码呢？
我第一次真正去读的源码是redux，它很小，很快就能看完。
看源码前和看源码后使用它是完完全全的两种感觉。
看源码之前，就是照着文档抄，在哪个文件哪个位置写哪个函数。
